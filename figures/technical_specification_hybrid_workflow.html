2.1 Problem Analysis: Why Matplotlib Fails

‚ö†Ô∏è Constraint Propagation Deficiency

Matplotlib's layout engine uses fixed coordinate systems (data, axes, figure coordinates). Complex hierarchical layouts require:

  1. Parent-child spatial relationships (encoder contains N layers)
  2. Dynamic label positioning (avoid overlap with variable text lengths)
  3. Arrow routing with orthogonal paths (not point-to-point Bezier curves)
  4. Consistent box sizing (equal width despite different label lengths)

Matplotlib's patches.FancyBboxPatch and FancyArrowPatch lack constraint-based layout algorithms present in Graphviz, draw.io, TikZ.

2.2 Hybrid Strategy: Scaffold ‚Üí Refine

Approach: Generate low-fidelity matplotlib version to establish data flow topology, then recreate in draw.io with precise control.

# fig1_architecture_scaffold.py import matplotlib.pyplot as plt import matplotlib.patches as mpatches from matplotlib.patches import FancyBboxPatch, FancyArrowPatch fig, ax = plt.subplots(figsize=(12, 10)) ax.set_xlim(0, 10) ax.set_ylim(0, 12) ax.axis('off') # Component definitions (x, y, width, height, label) components = [ # Input layer (1, 10, 2, 0.8, 'Raw Problem\nInput'), # RHINOCEROS layers (Pattern #1: Hierarchical) (1, 8.5, 2, 0.8, 'Context\nCompression'), (1, 7.0, 2, 0.8, 'PPWHH\nProtocol'), (1, 5.5, 2, 0.8, 'Reasoning\nEngine'), (1, 4.0, 2, 0.8, 'Generation\nEvolution'), # Parallel processing (4.5, 8.5, 2, 0.8, 'GPU Cluster\n(84 TOPS)'), (4.5, 7.0, 2, 0.8, 'Quantum\nValidator'), # Output (1, 2.0, 2, 0.8, 'Solution\nOutput'), ] # Draw boxes and arrows (simplified) for x, y, w, h, label in components: box = FancyBboxPatch((x, y), w, h, boxstyle="round,pad=0.1", edgecolor='#3b82f6', facecolor='#eff6ff', linewidth=2) ax.add_patch(box) ax.text(x + w/2, y + h/2, label, ha='center', va='center', fontsize=10, fontweight='bold', color='#1e3a8a') plt.savefig('fig1_architecture_scaffold.png', dpi=300, bbox_inches='tight')
